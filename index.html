3. Safety, Usage & Enhancement

Safety & Non-Destructive Routines

  • **Non-Root Operation:** The script uses standard Termux and Android commands (like `find`, `rm`, `ps`, `sync`), ensuring it cannot access or modify protected `/system` files.
  • **Targeted Deletion:** File cleanup is restricted to known temporary file types (`.tmp`, `.log`, cache folders) and specific user folders (`$STORAGE_PATH`).
  • **Logging:** All actions are logged to a dated file within `$HOME/apex_optimizer_logs` for transparency and error review.
  • **Service Management Alternative:** Since true service management is root-only, the script uses `ps` to kill "zombie" processes and `sync` to flush the kernel's memory buffer, safely reclaiming resources without altering system configurations.

Usage & Additional Features

  1. **Enable Storage:** Run `termux-setup-storage` once to grant the script access to your shared storage (`/sdcard`).
  2. **Save Script:** Copy the generated code above and save it in a file: `nano optimizer.sh`. Paste the code and save (Ctrl+X, Y, Enter).
  3. **Set Permissions:** Give the script execution rights: `chmod +x optimizer.sh`.
  4. **Run:** Execute the script: `./optimizer.sh`.
  5. **Automation:** For enhanced performance, install `termux-wake-lock` to keep Termux active and use cron jobs to run the script automatically every night: `pkg install cronie` then `crontab -e`. Add a line like: `0 3 * * * /data/data/com.termux/files/home/optimizer.sh` (Runs at 3 AM daily).
